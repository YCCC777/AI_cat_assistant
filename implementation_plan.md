# 貓咪助手（AI 考照小幫手）執行計畫與軟體規格書

## 1. 專案概述與目標
打造一個可愛、極簡、零成本維護的 LINE Bot。本專案不僅用於收集 AI 課程資訊並同步至 Google 行事曆與 Notion，更擴充為「AI 考照社群的專屬伴讀小管家」。透過結合 RSS 自動摘要的「AI 週報」與無痛管理的「學習卡進度機制」，陪伴使用者克服考前焦慮，建立每日學習習慣。

## 2. 目標使用者與使用情境
*   **一般 AI 資訊關注者**：平時想獲取免費 AI 課程資訊，或每週想輕鬆吸收 AI 大事件（AI 週報）。
*   **AI 證照考生（核心受眾）**：已報名或準備報名 AI 證照（如 iPAS AI 應用規劃師）的 Vibe Coder。

**使用情境：**
*   **日常通勤**：在捷運上點擊「餵罐罐」，花 3 分鐘看一張 AI 知識學習卡。
*   **考前衝刺**：頻繁點擊獲取新卡片，並點擊「捏肉球」查看倒數天數，獲取陪伴感。
*   **社群交流**：群友在 LINE 群組丟出優質 AI 新聞連結，系統自動攔截收錄。

## 3. 功能性需求（核心功能清單）
*   **【基礎】課程收集與同步**：解析 LINE 訊息中的課程資訊，同步至 Google 行事曆。
*   **【新增】AI 週報與社群收錄模組**：
    *   **自動摘要**：固定抓取指定 RSS 來源（如 iThome），利用 Gemini AI 摘要成週報。
    *   **社群共建**：自動攔截群組內帶有特定指令的外部文章連結，存入 Notion「待審核區」，每日設有收錄上限，防惡意洗版。
*   **【新增】專屬陪讀模組（學習卡機制）**：
    *   **報名宣誓**：首次啟動需設定考試項目（預設為 iPAS AI 應用規劃師 初級）與考試日期。
    *   **餵罐罐（學習卡）**：依據使用者個人進度，從 Notion 題庫發送「下一張」精華筆記。
    *   **學習卡跳過機制**：在每張學習卡文字訊息下方，附帶「換下一張（已熟悉此觀念）」的按鈕，讓使用者快速推進進度。
    *   **捏肉球（打卡倒數）**：計算並回覆距離考試的倒數天數及連續學習進度。

## 4. 使用者流程
1.  **加入好友 / 預設狀態**：使用者看到下方固定圖文選單（Rich Menu），包含「免費 AI 課程」、「AI 週報」、「貓咪陪讀專區」。
2.  **啟動陪讀**：點擊「貓咪陪讀專區」。若為首次點擊，機器人詢問：「是否為考生？請輸入要考的證照與日期」。使用者回覆後，機器人確認：「沒問題～本喵都記好了！」
3.  **日常陪讀互動**：設定完成後，再次點擊「貓咪陪讀專區」，機器人會回傳「輪播選單（Carousel）」，包含四個按鈕：
    *   **餵罐罐** -> 觸發派發下一張學習卡。
    *   **捏肉球** -> 觸發回報考試倒數天數與進度。
    *   **讀書進度** -> 查看目前已讀卡片數量總結。
    *   **陪讀設定** -> 重新設定或修改考試日期。
4.  **學習卡互動**：收到學習卡後，點擊下方附帶的「喵～我懂了（換下一張）」按鈕，機器人會傲嬌回覆並直接發送下一張卡片。
5.  **社群投稿**：使用者在群組輸入「/收錄 [網址]」，機器人回覆「喵～已放入待審核百寶箱！」。

## 5. 介面風格與平台
*   **平台**：LINE Messaging API。
*   **Persona 設定**：傲嬌但溫慢的「貓咪」，所有回覆必須帶有「喵～」且使用台灣繁體中文。具備魅力報錯台詞（如：「貪心會消化不良喵，明天再來！」）。
*   **UI 呈現**：採用底層「固定圖文大看板」搭配互動時彈出的「橫向滑動輪播卡片選單」，以及學習卡文字下方的快速按鈕。

## 6. 非功能性需求
### 6.1 效能需求（回應時間、並發量）
*   **考前防擠兌機制**：針對考前可能出現的「抱佛腳潮」（數百人同時點擊），系統需實作簡易快取（Cache）或請求排隊機制，避免 Notion API 超載。若發生擁塞，需優雅回覆：「喵～大家太熱情了，本喵的肉球忙不過來，請等 3 秒鐘再點一次喔！」

### 6.2 可靠性與可用性
*   部署於免費雲端服務（如 Render 或 Hugging Face），需容忍冷啟動時間。

### 6.3 安全性需求
*   **環境變數隔離**：API Keys（LINE, Google, Notion, Gemini）絕對不寫死在程式碼中。

### 6.4 資料隱私與合規
*   **佛系匿名制**：僅記錄 LINE 官方提供的無意義 User ID，不要求提供本名、電話或 Email，達到零個資外洩風險。

## 7. 資料需求
所有核心資料均依賴 Notion Database 進行無痛管理：
*   **課程資訊表（現有）**：存放解析出的課程時間與連結。
*   **使用者進度表（新增）**：記錄 LINE_User_ID（主鍵）、選擇考試、考試日期、目前卡片進度(Index)、最後互動日期。
*   **學習卡圖書館（新增）**：記錄 卡片編號、章節、知識點內容（純文字）。開發者只需在 Notion 打字即可擴充。
*   **AI 資訊待審核區（新增）**：記錄 投稿網址、收錄日期、核准狀態(Checkbox)。

## 8. 系統整合需求
*   LINE Messaging API (Webhook 接收、Rich Menu/Carousel Menu 綁定、Template Message 快速按鈕)
*   Google Calendar API (課程同步)
*   Notion API (題庫管理、進度儲存、文章審核區)
*   Gemini 2.5 Flash Lite API (課程資訊解析、RSS 摘要潤飾)
*   RSS 訂閱源 (如 iThome 或科技新報之公開 Feed)

## 9. 擴充性與未來規劃
*   **神級學霸模式（費曼學習法）**：未來考慮導入 LLM 扮演「85歲老奶奶」，讓使用者輸入自己的解釋以驗證觀念。
*   **錯題與間隔重複**：引入記憶曲線機制（Spaced Repetition），答錯的題目三天後自動重考。
*   **開源社群 DIY 擴充**：於開源文件中提供教學，指導進階使用者如何利用 Dify/Coze 免費串接這些高級功能。

## 10. 優先順序與分期計畫
*   **第一期**：完成基礎架構、LINE/Google/Notion 串接與課程解析（已完成）。
*   **第二期（本次擴充）**：實作 Notion 學習卡庫、Rich Menu 動線、餵罐罐進度機制與防擠兌限流。
*   **第三期**：實作 RSS 自動抓取 AI 週報與社群網址投稿審核機制。
